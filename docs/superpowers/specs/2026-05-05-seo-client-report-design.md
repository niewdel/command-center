# SEO Client Report вҖ” Townsquare-style V1

**Date:** 2026-05-05
**Status:** Approved for implementation
**Owner:** Justin Ledwein

## Goal

Restructure the per-client SEO report so it matches the Townsquare Interactive
report aesthetic (dense card grid, big cyan numbers on dark navy, date range
filter, current-vs-lifetime comparisons) using only data we already collect.
Apply the same rendering to two surfaces:

1. The in-app client dashboard at `/seo/clients/[id]`
2. The monthly PDF emailed to clients

A single React renderer drives both. The PDF is generated by pointing
Playwright at the in-app report URL with `?print=1`, eliminating the parallel
HTML template.

## Non-goals (V1)

- New integrations (GBP reviews, call tracking, form-fill capture, marketing
  email/SMS metrics). Sections that need them are deferred until the
  integration ships.
- Device split (desktop/mobile traffic). Not in `seo_traffic_snapshots`
  schema today; trivial follow-up by extending the GA4 query.
- `6m / YTD / 1y` date range tabs. V1 ships `30d / 90d / Lifetime`.
- Multi-tenant theming hooks. Tokens already live in CSS variables, so this
  is incidentally future-proofed, but no per-tenant overrides ship now.

### Data fetcher

`src/lib/seo/report-data.ts` exports:

```ts
type ReportRange = "30d" | "90d" | "life";

interface ReportData {
  client: { id, name, domain, period_label, generated_at };
  range: ReportRange;
  health: {
    overall_score: number | null;          // weighted avg, 0-100
    overall_delta: number | null;
    technical: ScoreCard;
    onpage: ScoreCard;
    lighthouse_mobile: ScoreCard;
    lighthouse_desktop: ScoreCard;
    open_issues: { total, critical, high, medium, low };
  };
  traffic: {
    sessions: MetricCard;
    organic_sessions: MetricCard;
    users: MetricCard;
    pages_per_session: MetricCard;
    sources: { search, direct, referral, social, other }; // percentages
  } | null;                                 // null when GA4 not configured
  keywords: {
    ranking_count: number;                  // phrases with non-null rank
    tracked_count: number;                  // total tracked
    avg_rank: number | null;
    total_search_volume: number;
    top_movers_up: KeywordMover[];          // up to 5
    top_movers_down: KeywordMover[];        // up to 5
  } | null;                                 // null when no keyword data
  top_pages: Array<{ path, sessions, pct_of_total }>; // up to 5
  issues: {
    open_top: SeoIssueRow[];                // top 10 critical+high open
    resolved: SeoResolvedRow[];             // resolved in window
  };
  history: ScoreHistoryPoint[];             // for chart
  ai_summary: string | null;
}

async function getReportData(
  client_id: string,
  range: ReportRange
): Promise<ReportData>
```

The fetcher narrows seo_checks/issues/keyword_ranks/traffic_snapshots to the
window and computes deltas against the row at the start of the window. Lifetime
range disables window filtering.

### Print mode security

`?print=1` bypasses session auth via a signed token query param:
`?print=1&token=<HMAC(client_id + range + day_bucket)>`.

The token uses an env secret (`SEO_REPORT_PRINT_SECRET`), is valid for the day,
and can only access the report route. Playwright signs the token in
`monthly-report.ts` before navigating. No other route accepts this token.

## Sections (V1)

1. **Header** вҖ” client name, domain, range tabs (30d/90d/Lifetime), last
   updated, "Generate Report" button (in-app only)
2. **SEO Health** вҖ” Overall Score (hero) + Open Issues (hero) + 4-up
   technical/onpage/mobile/desktop with sparklines + deltas
3. **Site Traffic** вҖ” Sessions (hero) + 3-up organic/users/pages-per-session
   + Traffic Sources strip (% breakdown). Hidden when traffic is null.
4. **Keyword Rankings** вҖ” "X/Y phrases ranking" + avg rank + search volume +
   top movers (up + down). Hidden when keywords is null.
5. **Top Pages** вҖ” Top 5 GA4 pages with % of total. Hidden when traffic is
   null.
6. **Issues & Wins** вҖ” open critical/high left, resolved right
7. **Score Trends** вҖ” existing `<ScoreHistoryChart>` over the range
8. **AI Summary** вҖ” `seo_checks.ai_summary` prose
9. **Footer** вҖ” Niewdel wordmark + generation date + range label

Sections that depend on optional data (traffic, keywords) hide cleanly when
the data is absent вҖ” no empty placeholders. For example, a client with no
GA4 connection sees Health вҶ’ Keywords вҶ’ Issues вҶ’ Trends, no broken cards.

## Visual

- **Surfaces:** `bg-background` page, `bg-card` cards with 1px
  `border-border`. No gradients, no colored shadows (baseline rules).
- **Numbers:** `text-primary` (electric cyan), `font-data` for tabular nums.
  Hero metrics `text-5xl font-semibold`, secondaries `text-3xl`.
- **Labels:** `text-muted-foreground text-xs uppercase` вҖ” no
  `tracking-*` modifiers (baseline rules).
- **Deltas:** green for positive, `text-destructive` for negative, with
  `вҶ‘`/`вҶ“` glyph and `tabular-nums`.
- **Severity pills:** reuse existing color mapping from issue UI.
- **Cards:** `rounded-lg p-6`. No hover glow.
- **Date range tabs:** existing shadcn `Tabs` primitive. Active state
  `bg-foreground text-background` per baseline.
- **Print mode (`?print=1`):** strips sidebar + admin chrome, sets
  `@page { margin: 0 }`, footer template via Playwright.

## Date range mechanics

| Range | Window for seo_checks / issues / keywords | GA4 traffic |
|-------|-------------------------------------------|-------------|
| `30d` | created_at >= now() - 30d | most recent snapshot |
| `90d` | created_at >= now() - 90d | most recent snapshot |
| `life` | no filter | most recent snapshot |

Deltas always compare the latest row in the window to the row at the *start*
of the window (or earliest for `life`). Traffic deltas keep using the prior
snapshot вҖ” that's already how it works and the snapshot's own period bounds
are the natural comparison.

## Acceptance criteria

- [ ] `/seo/clients/[id]/report?range=30d` renders all V1 sections for a
      client with full data (health + traffic + keywords + issues).
- [ ] Same route renders cleanly for a client with no GA4 (traffic + top
      pages sections hidden).
- [ ] Same route renders cleanly for a client with no keyword tracking
      (keywords section hidden).
- [ ] Range tabs change query param and re-render with the right window.
- [ ] `?print=1&token=вҖҰ` route renders without sidebar, accepts only valid
      tokens, and Playwright produces a PDF byte-identical in layout to the
      in-app view (same components).
- [ ] Monthly cron produces a PDF and uploads to the existing
      `audit-reports` bucket using the new flow.
- [ ] `monthly-report-html.ts` is deleted.
- [ ] `seo/clients/[id]/page.tsx` is under 1000 lines and shows admin
      chrome + embedded report.
- [ ] No baseline UI rule violations (no gradients, glow shadows,
      `tracking-*`, `h-screen`, etc.).
